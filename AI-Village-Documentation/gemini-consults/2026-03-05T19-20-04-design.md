## 2. Responsive Behavior (10 Breakpoints)

We must maintain absolute control across all 10 breakpoints. 

*   **320px (Small Mobile)**: Single column. Image height `180px`. Padding `16px`. Button spans 100% width.
*   **375px / 430px (Standard/Large Mobile)**: Single column. Image height `240px`. Padding `24px`. Meta data displays in a 2x2 grid.
*   **768px (Tablet Portrait)**: Reflows to a Row layout. Image left (40% width), Content right (60% width). Button becomes inline-flex.
*   **1024px (Tablet Landscape / Small Desktop)**: Padding increases to `40px`. Title font size scales to `40px`.
*   **1280px / 1440px (Standard Desktop)**: Padding `48px`. Title `48px`. Hover interactions fully enabled.
*   **1920px (HD Desktop)**: Card locks to `1200px` max-width and centers.
*   **2560px / 3840px (2K/4K Ultrawide)**: Typography scales up using `clamp()` functions (handled in CSS) to prevent the UI from looking microscopic.

## 4. Accessibility

*   **ARIA**: Card must act as a `<section>` with `aria-labelledby="workout-title"`.
*   **Focus Management**: Custom focus rings. No default browser outlines.
    *   `:focus-visible { outline: 2px solid #00FFFF; outline-offset: 4px; }`
*   **Contrast**: `#B4B4C8` on `rgba(10, 10, 26, 0.6)` yields a contrast ratio of ~7.5:1 (WCAG AAA compliant).
*   **Reduced Motion**: Wrap all animations in `@media (prefers-reduced-motion: reduce) { transition: none; animation: none; }`.

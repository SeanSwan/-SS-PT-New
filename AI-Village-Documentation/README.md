## üö® Important Rules

### NEVER Skip the Pipeline
- **Every feature** must pass all 7 checkpoints (or approved Fast Track variant)
- 30 minutes in pipeline saves 2-4 hours debugging production
- Zero exceptions for "quick fixes"

### Pipeline Variants
- **FULL** (all 7): New features, security code, DB changes
- **FAST TRACK** (1,2,4,7): UI polish, minor bug fixes, styling
- **CRITICAL PATH** (1,2,3,4,6,7): Security patches, data privacy

### Git Push Rules
- Only push if all checkpoints = ‚úÖ PASS
- Include "Passed all 7 AI review checkpoints" in commit message
- If ANY checkpoint fails ‚Üí fix ‚Üí re-run from that checkpoint

## üÜò Getting Help

### If Stuck on a Checkpoint
1. Read the AI's feedback carefully
2. Fix the specific issue mentioned
3. Re-run from THAT checkpoint (not from #1)
4. If stuck after 3 attempts ‚Üí ask Claude Desktop for architectural guidance

### If Tools Are Down
- **Roo Code down?** ‚Üí Use Codex (GPT-4 in VS Code) as fallback
- **OpenRouter down?** ‚Üí Use Claude Code for backend coding
- **Gemini down?** ‚Üí Use v0.dev for frontend, then Claude Code for integration

### If You Find a Bug in Production
1. Don't panic - even with AI Village, you'll still catch 95%+ (not 100%)
2. Create hotfix branch
3. Run FULL pipeline even for small fix
4. Post-mortem: "Which checkpoint should have caught this?"
5. Update that checkpoint's prompt
